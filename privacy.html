Privacy and data protection

How we safeguard personal information and keep our customers GDPR-compliant

Where we handle personal information, for example data relating to the indoor environment within someone's home, we ensure that their privacy is protected using pseudonymisation.

  • Pseudonymisation means ensuring that data from a sensor or a meter is tagged with a code that is unrelated to the property address. So for example temperature data from a bedroom at 99 Acacia Avenue is tagged with a device address like ab:23:cd:45:ef:67 so that anyone obtaining access to that data cannot deduce which dwelling it relates to unless they also have access to a table which links the address to the tag. This table is kept securely in the cloud and can only be accessed by those with a business need. Installers, who inevitably know which device goes into which property, are not permitted to see the data.
  • GDPR (the UK GDPR and the Data Protection Act 2018) places obligations of lawful processing, transparency, data minimization, and enabling tenant rights on social housing providers. Pseudonymisation allows us to process data lawfully by ensuring that the personal nature of the data we gather is removed when the data is in transit and is only restored within the housing provider's secure cloud environment.
